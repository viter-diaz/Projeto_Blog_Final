# Projeto Blog Final

## 📌 Descrição

Este projeto foi desenvolvido como parte do desafio **Utilizando Recursos do GitHub em um Projeto Open Source**, da DIO.

O objetivo do desafio é aplicar na prática os principais recursos do Git e GitHub, criando ou melhorando um projeto, versionando o código e organizando a documentação em um repositório público.

## 🚀 Objetivo

Praticar o uso de recursos essenciais do GitHub em um projeto, como:

* Criação de repositório público
* Versionamento com Git
* Commits
* Branch principal `main`
* Envio do projeto para o GitHub
* Documentação com Markdown

## 🛠️ Tecnologias Utilizadas

* Git
* GitHub
* Markdown
* HTML
* CSS
* JavaScript

## 📂 Estrutura do Projeto

```bash
Projeto_Blog_Final/
│
├── index.html
├── README.md
├── assets/
├── css/
└── js/
```

## 🔧 Comandos Utilizados

```bash
git init
git add .
git commit -m "Primeiro commit do projeto"
git branch -M main
git remote add origin https://github.com/viter-diaz/Projeto_Blog_Final.git
git push -u origin main
```

Caso o GitHub já tenha arquivos no repositório, foi necessário sincronizar com:

```bash
git pull origin main --allow-unrelated-histories
git push -u origin main
```

## 📚 Aprendizados

Durante este desafio, foram praticados conceitos importantes como:

* Como criar um repositório no GitHub
* Como conectar um projeto local ao GitHub
* Como resolver conflitos iniciais entre repositório local e remoto
* Como organizar commits
* Como documentar um projeto usando README.md
* Como publicar um projeto no GitHub
